## 決定事項・実施内容

### 1. 開発環境方針の決定

- **Bun はローカルインストールしない**（ユーザー要望）
- すべての Bun コマンドは `docker run oven/bun:1` 経由で実行する
- 開発サーバーは root の `compose.yaml` で `docker compose up` により起動する
- `make` コマンドで全操作をラップ（Makefile 作成）

### 3. Cloudflare Workers デプロイ実装

- **D1 アダプタ**（`src/adapters/repository/d1.ts`）: raw D1 API で実装（`@cloudflare/workers-types` の型競合を回避するためローカル型定義を使用）
- **R2 アダプタ**（`src/adapters/storage/r2.ts`）: R2 Bucket binding API で実装
- **Worker エントリ**（`src/entries/worker.ts`）: D1/R2/ASSETS binding を組み上げ。`/static/*` は Workers Assets にオフロード
- `deploy/cloudflare/wrangler.toml`: D1/R2/Workers Assets/カスタムドメイン設定
- `deploy/cloudflare/migrations/0001_init.sql`: D1 スキーママイグレーション

### 4. トラブルシューティング記録

| 問題 | 原因 | 解決策 |
|---|---|---|
| `bun run typecheck` エラー | `c.body(Uint8Array)` 型不一致 | `r.data.buffer as ArrayBuffer` にキャスト |
| DB 起動エラー | データディレクトリが未作成 | `container.ts` で `mkdirSync` を追加 |
| Cloudflare API Token 無効 | `/user/tokens/verify` がアカウントスコープトークンに非対応 | `/accounts/{id}/tokens/verify` を使用。トークン自体は有効だった |
| wrangler が D1 作成で認証エラー | Token 権限が不足（Workers Routes Write のみ） | D1 Edit / R2 Storage Edit / Workers Scripts Edit / Account Settings Read の4権限に変更 |
| R2 バケット作成エラー | R2 が未有効化 | Cloudflare ダッシュボードから R2 を有効化 |
| wrangler deploy がハングする | `bunx wrangler` が Bun 上で正常動作しない（非同期処理の互換性問題） | `node:20-slim` + `npx wrangler@4` に切り替え |
| CSS が適用されない | Workers Assets の `directory = dist/static` のためファイルパスが `/style.css` になっていた | `directory = dist` に変更し `/static/style.css` で配信されるよう修正 |
| `routes` が d1_databases に吸収される | TOML の配列セクション内に誤配置 | `routes` をファイル先頭付近（`[[...]]` より前）に移動 |
| カスタムドメインのパターンエラー | `pagebox.iodine2.net/*` はワイルドカード不可 | `pagebox.iodine2.net`（パスなし）に修正 |
